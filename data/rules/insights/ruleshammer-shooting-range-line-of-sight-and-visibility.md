Welcome back to Ruleshammer! In this series we take a detailed look at the rules of key games, but Warhammer 40k in particular. This week I’m going through a few of the subtleties for Shooting in 11th edition, looking in detail at Range, Line of Sight, and Visibility.

Shooting in Warhammer 40k 11 edition has a few changes and updates from previous editions that might surprise new and veteran players alike, let’s dive in.

## SELECT TARGETS 04.02

The source of one of the most subtle changes in the edition happens here at step 2 of the attack sequence.

WHILE SHOOTING

Select one enemy unit to be the target of that weapon. Unless otherwise stated, each target must be:

  - Visible to the model that has that weapon (06.01).
  - Within range of that weapon.
  - Unengaged
Unlike previous editions, the model that is visible and the model that is in range of a specific weapon **no longer need to be the same model**. It’s a very subtle change, so much so that they already FAQed to confirm this is intended.

**Q.** Can a model be within range of a weapon if it is not visible?

**A.** Yes

## What does this mean for Rapid Fire, Melta, etc?

They continue to work as they have done for a while, but without being a slightly odd exception that works differently to how weapon range and visibility checks work. MELTA is concerned with how far away the unit is, not how far away a visible model in the target unit is.

This ability always takes the form \[MELTA X\]. Each time a model makes an attack with a \[MELTA\] weapon, if the target unit was within half range of that weapon in the Select Targets step, until the attacking unit’s attacks have been resolved, add X to that weapon’s D characteristic.

And so is RAPID FIRE.

This ability always takes the form \[RAPID FIRE X\]. Each time you gather attack dice for a \[RAPID FIRE\] weapon, add X additional attack dice if the target unit was within half range of that weapon in the Select Targets step.

Note also that this is checked at the “select targets step” so even if you have a unit with several weapons, and only some of them have the MELTA ability, if you resolve the none-MELTA weapons first that does not prevent the MELTA ability from triggering if the unit was within half range when you selected targets.

## Line of Sight and Visibility Checks 06.01

*EDIT: This section has been amended since publication based on feedback, there is some debate on how this section should be interpreted and I think it might require an FAQ.*

One aspect I feel is currently left slightly unclear is how to check if a model is fully visible or not. We've been given a technical definition for a Line of Sight but not specifically told when to use it, other than to determine if models are purely visible or not.

Line of sight is used to determine visibility between models. For an observing model to have line of sight, it must be possible to draw an imaginary straight line, 1 mm wide, from any part of that model to any part of the model being observed. This line is the line of sight. While doing so, other models in the observing model’s unit and in the observed model’s unit are ignored.

Other models and units can be either visible or fully visible to the observing model, as shown opposite. Note that terrain applies additional rules to visibility (13.07).

The rules then go say that visible or fully visible is defined by the set of example images on page 25 of the Core Rules. We've mulled this over around the TTB water cooler and we're still not sure (or at least I'm not fully convinced). The text around the "fully visible" example image potentially implies that lines are intended to be imagined.

If every part of another model that is facing the observing model **is visible** to the observing model (so the only thing blocking visibility to any part of that other model is that model itself), that model is fully visible.

As some of us think that to determine if a part "is visible" you follow the rule as written above, imagining lines from the attacking model to every part of the target. In my opinion this doesn't fully line up with the example images but they could just be bad examples, and it also leaves the open question of if all of these lines need to be originate from one part of the attacking model, of if they can originate from many parts.

## Can Models block Visibility to other models?

Technically it’s possible though very few models are completely opaque in this way, you can often shoot under most tanks even if they’re on wheels or tracks. My advice would generally be to presume such models won’t block Line of Sight and to check with your opponent if you think one should or currently is blocking a model from view.

Relatedly though whilst models in other units can make a model not fully visible, being not fully visible doesn’t do anything on its own. You need another rule triggered by being not fully visible such as the Benefit of Cover 13.08 rule provided by terrain features. There are some models that have rules like this, the Baneblade for instance can provide cover to models not fully visible due to its bulky frame. The smokescreen stratagem causes the model you use it on to provide cover also.
