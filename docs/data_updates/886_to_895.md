# w40k data update: 886 to 895

Date: 2026-07-15
Kind: data update
Versions: 886 -> 895
Summary: Balance pass. Veiled Blade assassin points up, 13 new Space Marines / Black Templars leader groups, Tyranids Frame keyword, one new core rule, 2 new FAQs.

Date swapped: 2026-07-15. Source: official Warhammer 40,000 app `base.apk`
(pulled 2026-07-15), exported with `scripts/w40k_exporter/w40k_exporter.py`
per `CODEX_ARMORUM_DATA_UPDATE.md`. This file records every change between
data_version 886 and data_version 895, derived from a field-level diff of the
old and new `w40k.db` (summary compare via `scripts/compare_w40k_db.py`, then
a full-table row diff; no table outside the sections below changed).

Headline: a small balance pass. No schema changes, no factions added or
removed, no datasheets added or removed, no detachment/battle-size/keyword
changes. Everything below is content-level.

## 1. Points changes (4 datasheets)

Only the Veiled Blade Elimination Force detachment tier moved; the base
Agents of the Imperium tier and the untagged (allied) tier are unchanged.

| Datasheet [Imperial Agents] | Tier | 886 | 895 |
|---|---|---|---|
| Callidus Assassin | Veiled Blade Elimination Force | 100 | 140 |
| Culexus Assassin | Veiled Blade Elimination Force | 85 | 125 |
| Eversor Assassin | Veiled Blade Elimination Force | 110 | 145 |
| Vindicare Assassin | Veiled Blade Elimination Force | 110 | 155 |

Unchanged for reference: Callidus 100/100, Culexus 85/85, Eversor 110/120,
Vindicare 110/125 (Agents tier / allied tier).

## 2. Leader attachments (leader_group 1267 to 1280, +13 groups)

All 13 new groups are Space Marines / Black Templars ecosystem:

- The Black Templars' own **Sternguard Veteran Squad** datasheet
  (`cec275e8-0bf3-499c-9646-3b84443eb02c`) is now attachable for the generic
  Space Marines characters **Ancient, Apothecary, Captain, Chaplain,
  Judiciar, Lieutenant** (6 groups). Previously only Castellan, Emperor's
  Champion and Marshal could lead it. Each affected character's `leads_units`
  now lists "Sternguard Veteran Squad" twice (the Space Marines datasheet and
  the Black Templars datasheet share a name).
- **Castellan, Emperor's Champion, Marshal** each gained a *duplicate* second
  group for that same Black Templars Sternguard datasheet (3 groups). This is
  redundant data in the source app, not a rules change.
- **Eradicator Squad with Heavy Bolters** is now leadable at all: new groups
  for **Apothecary Biologis, Captain in Gravis Armour, Iron Father Feirros,
  Tor Garadon** (4 groups). Its `can_be_led_by` went from empty to those four.

## 3. Other datasheet content changes

- **Amonhotekh's Guard Canoptek Doomstalker** [Necrons]: lost its
  **Reanimation Protocols** faction ability row (this is the ability table's
  net -1, 4310 to 4309).
- **Kairos Fateweaver** [Scintillating Legions datasheet
  `4d987664-7ac1-448b-9cd4-715be7646fd3` only]: **Infernal Gateway** focused
  witchfire profile attacks changed **D3+6 to D6+6**; profile names
  recapitalised ("focused witchfire" to "Focused Witchfire", "witchfire" to
  "Witchfire"). The source `wargear_loadout.default_loadout` array was also
  emptied (options and choose_from unchanged; our engine still derives a
  legal default, engine invariants pass). Note: his second datasheet under
  **Legiones Daemonica** (`02f35395`) was NOT updated and still carries D3+6;
  that inconsistency is in the official app's own data, verified in both the
  db and our rendered API.
- **Toxicrene, Tyrannocyte, Tyrannofex** [Tyranids]: gained the **Frame**
  keyword (unit and model keyword lists). This bakes in what the existing
  "The following units have the FRAME keyword" errata FAQ already said.

## 4. Stratagem id rotation (no content change)

The five "renames" the summary compare reports are the same five stratagems
with their database ids rotated; the set of (name, rules text) pairs is
identical in 886 and 895:

- Leagues of Votann, Armoured Trailblazers: ids 1124/1125/1126 rotated
  between Built To Last, Coordinated Crossfire, Outflanking Armour.
- Necrons, Hand of the Dynasty: ids 1344/1345 swapped between Nanosaturation
  and Will of the Conqueror.

Nothing in Codex Armorum references stratagem row ids, so this is a no-op.

## 5. Enforcement-data changes (army-builder relevant)

- **allied_faction.allowed_warlord_miniatures emptied** on all three rows
  that carried data: Agents of the Imperium (Imperium hosts row), Agents of
  the Imperium (Deathwatch host row), Heretic Astartes in Legiones Daemonica.
  Each went from a 8-10 miniature whitelist to `[]`. The builder does not yet
  consume this field (ally warlord rules are an open parity-gap item), so no
  behaviour change today; noted for when that lands.
- **enhancement.grants_leader_attachment emptied** for **Murdermind**
  (Necrons, Hypercrypt/Destroyer Cult; previously granted attachment to
  Lokhust Destroyers, Lokhust Heavy Destroyers and others) and **Exemplar of
  Duty (Upgrade)** (Astra Militarum; previously Ogryn Squad, Bullgryn Squad).
  Grant groups drop 19 to 13; the engine loads and enforces this data, so
  those bearers no longer gain extra attach targets, matching the official
  app.
- `allied_faction.datasheets` lists re-ordered on the same three rows (same
  membership, order only).

## 6. Core rules, FAQs, missions

- **New rule 09.05.01** under Normal Move: "A unit cannot make more than one
  Normal Move in a phase." Merged into
  `data/rules/wh40k_core_rules_combined.md` and rebuilt.
- **New rule section "Deployment Card Key"** (rule_section 33 to 34,
  rule_block +20 with 09.05.01): an icon legend for mission deployment cards.
  Eight icons (CDN-hosted, not exported as files) with captions defining
  Attacker's/Defender's Battlefield Edge, Deployment Zone and Territory, No
  Man's Land, and the Home/Central/Expansion Objective icons. Not merged into
  /rules (it is a mission-card legend keyed to images the export does not
  carry; the /missions page already renders the layouts themselves). Text
  recorded here so nothing is lost.
- **Masters of Shadow** (Shadowmark Talon detachment) rule text errata'd.
  Old: "... the target has the Benefit of Cover against that attack."
  New: "... the target has the **Benefit** **of Cover**." The matching FAQ
  errata entry was replaced: the old FAQ's wording additionally imposed
  "subtract 1 from the hit roll", which the new FAQ drops entirely.
- **New FAQs (+2 net: 3 added, 1 replaced-out)**: "The Grenade Stratagem is
  the same as the Explosives stratagem"; "The Tank Shock Stratagem is the
  same as the Crushing Impact stratagem"; plus the Shadowmark Talon
  replacement above.
- **FAQ references (946 to 957, +11)**: Hive Crone newly linked to the
  "units lose the Hover core ability" errata; duplicate reference rows added
  for the existing Tyranids errata (Harpy, Harridan, Hierophant, Hive Crone,
  Toxicrene, Tyrannocyte, Tyrannofex each now reference their errata twice;
  duplicate rows in the source app, harmless). FAQ display is driven straight
  from `w40k.db`, so these flow through with no manual step.
- **Outflank** secondary objective (Chapter Approved pack): typo fix in the
  5VP criteria, "is not within 5VP your territory" to "is not within your
  territory".
